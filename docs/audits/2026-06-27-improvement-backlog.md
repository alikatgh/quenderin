### 1. Icon-only voice/mic button has no accessible name
`ui/src/components/GeneralChatArea.tsx:435-442` — a11y · high/S

**Why:** WCAG 2.1 AA 4.1.2 (Name, Role, Value): a <button> whose only content is an SVG icon exposes no accessible name, so screen-reader users hear just "button". The mic is the primary voice-input control. Adding aria-label (and type="button") fixes the name and prevents accidental form submission, without changing behavior. aria-pressed also communicates the recording toggle state.

**Change:**
```
Before:
<button
    onMouseDown={startRecording}
    onMouseUp={stopRecording}
    onMouseLeave={isRecording ? stopRecording : undefined}
    className={`p-2 rounded-lg transition-all ${isRecording ? 'bg-red-500 text-white animate-pulse' : 'text-zinc-400 hover:text-zinc-600 dark:text-zinc-500 dark:hover:text-zinc-300 hover:bg-zinc-100 dark:hover:bg-zinc-800'}`}
>
    <Mic className="w-4.5 h-4.5" />
</button>

After:
<button
    type="button"
    aria-label={isRecording ? 'Stop voice recording' : 'Start voice recording'}
    aria-pressed={isRecording}
    onMouseDown={startRecording}
    onMouseUp={stopRecording}
    onMouseLeave={isRecording ? stopRecording : undefined}
    className={`p-2 rounded-lg transition-all ${isRecording ? 'bg-red-500 text-white animate-pulse' : 'text-zinc-400 hover:text-zinc-600 dark:text-zinc-500 dark:hover:text-zinc-300 hover:bg-zinc-100 dark:hover:bg-zinc-800'}`}
>
    <Mic className="w-4.5 h-4.5" />
</button>
```

### 8. Modal has no Escape-to-dismiss and no focus management
`ui/src/components/TroubleshooterGuide.tsx:341-379` — a11y · high/M

**Why:** The element declares role="dialog" aria-modal="true" but never moves focus into the dialog on open and provides no Escape handler. WCAG 2.1.2 (No Keyboard Trap) and the ARIA dialog pattern require Escape to close and initial focus to land inside. A troubleshooter whose entire purpose is dismissal must be keyboard-dismissable. Escape is the single most expected key for a modal and is currently a no-op.

**Change:**
```
Add a keydown handler + initial focus. At the top of the component (after the `if (!action) return null;` guard is fine since hooks must be unconditional — place the effect above the early return):

  import { useState, useEffect, useRef } from 'react';
  ...
  const dialogRef = useRef<HTMLDivElement>(null);
  useEffect(() => {
    if (!action) return;
    dialogRef.current?.focus();
    const onKey = (e: KeyboardEvent) => { if (e.key === 'Escape') onResolved(); };
    document.addEventListener('keydown', onKey);
    return () => document.removeEventListener('keydown', onKey);
  }, [action, onResolved]);

Then on the inner panel div (line 343) add ref + tabIndex so it can receive focus:
  <div ref={dialogRef} tabIndex={-1} className="bg-white dark:bg-[#18181b] w-full max-w-lg rounded-2xl ... focus:outline-none" >

(Keep the hook above `if (!action) return null;` to satisfy the Rules of Hooks; guard inside the effect with `if (!action) return;`.)
```
