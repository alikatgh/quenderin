# r11 — Quenderin a11y audit (2026-07-11)

**Lens:** WCAG 2.1 AA — keyboard, screen readers, focus, touch targets (50-round plan r11)
**Mode:** Inline audit + same-session fixes (Claude).

## State of play (verified good — prior work)
- Icon-only controls carry `aria-label` (mic, stop, send, sidebar toggle, dismiss); record button
  has `aria-pressed`; Inspector toggle has `aria-expanded`; nav uses `aria-current="page"`.
- Toggles are real `role="switch"` + `aria-checked`; wizard progress is a labeled `progressbar`.
- `prefers-reduced-motion` globally respected (`ui/src/index.css:162`) + `motion-safe:`/`motion-reduce:` usage.
- Voice buttons use pointer events (touch works); PrivacyLock autofocuses and announces via `role="status"`.

## Findings

### A1 — Chat transcripts had no live region — **FIXED** (High)
- **Files:** `ui/src/components/GeneralChatArea.tsx:197`, `ui/src/components/ChatArea.tsx:79`
- **Symptom:** Streamed replies and agent activity appended visually but were never announced —
  the core surface of the app was silent to screen readers (TasksArea already had `role="log"`;
  the two chat views didn't).
- **Fix:** `role="log"` + `aria-label` ("Conversation" / "Agent activity") on both transcript
  containers; log additions announce politely.

### A2 — WelcomeWizard never took focus — **FIXED** (Medium)
- **File:** `ui/src/App.tsx` (WelcomeWizard)
- **Symptom:** `role="dialog" aria-modal` over a `pointer-events-none` app, but focus stayed
  behind the backdrop — keyboard/SR users had no way to discover the wizard.
- **Fix:** Panel gets `tabIndex={-1}` + focus on mount (same pattern as TroubleshooterGuide).

### A3 — New error affordances are announced — **FIXED WITH r10**
- U2/U3 error states carry `role="alert"` / render inside labeled sections; Retry buttons are
  real buttons with visible text.

## Open (logged, not fixed here)
- Full focus *trap* (wrap-around Tab) in WelcomeWizard/TroubleshooterGuide is not implemented —
  initial focus + aria-modal only. Low practical impact (modals are short), candidate for a
  shared `useFocusTrap` hook if a future round adds more dialogs.
- Contrast spot-checks passed on the zinc palette, but no automated contrast pass was run —
  candidate for axe-core in CI in a later round.
